### 🎨 Phase 1: UI 텍스트 변경 (안전)
**예상 소요시간**: 2-3시간  
**위험도**: 🟨 Medium

#### 📝 변경 대상
1. **사용자 인터페이스 텍스트**
   - 페이지 제목, 헤더, 푸터
   - 메타데이터 (title, description)
   - 로고 alt 텍스트

2. **문서 파일**
   - README.md, CLAUDE.md
   - docs/ 폴더 내 모든 문서
   - 지시서 템플릿

#### ⚠️ 주의사항
- **변수명/함수명은 건드리지 않음**
- **파일 경로는 유지**
- **API 엔드포인트는 그대로**

---

### 🔀 Phase 2: API 병행 지원 (점진적)
**예상 소요시간**: 4-5시간  
**위험도**: 🟧 High

#### 🛠️ 구현 전략
1. **기존 API 유지** + **새 API 추가**
   ```
   기존: /api/youtube-lens/* (유지)
   새로: /api/trend-finder/* (추가)
   ```

2. **점진적 전환**
   - 새 컴포넌트는 새 API 사용
   - 기존 컴포넌트는 기존 API 유지
   - 사용자에게 무중단 서비스

#### 🎯 변경 대상
- `src/app/api/youtube-lens/` → 새 엔드포인트 복사
- React Query 훅 업데이트
- 클라이언트 코드 점진적 전환

---

### 🗃️ Phase 3: 데이터베이스 마이그레이션 (Critical)
**예상 소요시간**: 2-3시간  
**위험도**: 🟥 Critical

#### 📊 테이블 변경 계획
| **기존 테이블명** | **새 테이블명** | **방법** |
|------------------|-----------------|----------|
| `youtube_lens_*` | `trend_finder_*` | CREATE + 데이터 복사 |
| `yl_*` | `tf_*` | CREATE + 데이터 복사 |

#### 🛡️ 안전 전략
1. **새 테이블 생성** (기존 테이블 유지)
2. **데이터 복사**
3. **검증 완료 후** 기존 테이블 삭제

#### ⚠️ Critical Point
- **데이터 손실 방지**가 최우선
- **트랜잭션** 단위로 처리
- **롤백 시나리오** 항상 준비

---

### 📁 Phase 4: 파일/폴더 구조 변경 (Critical)
**예상 소요시간**: 5-6시간  
**위험도**: 🟥 Critical

#### 🗂️ 변경 대상
```
src/components/features/tools/youtube-lens/
→ src/components/features/tools/trend-finder/

src/lib/youtube-lens/
→ src/lib/trend-finder/

src/app/(pages)/tools/youtube-lens/
→ src/app/(pages)/tools/trend-finder/
```

#### 📝 변경 순서
1. **새 폴더 생성** + **파일 복사**
2. **import 경로 업데이트**
3. **테스트 검증**
4. **기존 폴더 삭제**

---
