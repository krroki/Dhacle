# 📚 Reference - 빠른 참조 가이드

## 📌 문서 관리 지침
**목적**: 참조 문서 디렉터리 가이드 - 빠른 데이터 조회를 위한 종합적인 룩업 테이블 제공
**대상**: 모든 AI (구체적인 정보나 현재 상태 확인이 필요한 경우)
**범위**: 참조 문서 사용법과 연계 패턴만 포함 (구현 단계나 해결책 없음)
**업데이트 기준**: 하위 reference 문서 추가/제거 시 즉시 업데이트
**최대 길이**: 20000 토큰 (현재 약 18000 토큰)
**연관 문서**: [빠른 시작](../tutorial/01-quick-start.md), [실무 가이드](../how-to/README.md)

## ⚠️ 금지사항
- 구체적인 구현 방법이나 해결 단계 추가 금지 (→ how-to/ 문서로 이관)
- 상세한 기술 설명이나 튜토리얼 내용 추가 금지 (→ tutorial/ 문서로 이관)
- 배경 지식이나 설계 철학 추가 금지 (→ explanation/ 문서로 이관)

## 📋 Reference 목록

### 📊 [프로젝트 현재 상태](project-status.md)

- 영역별 검증 명령어 (TypeScript, 린트, 빌드, 테스트)

- 성능 측정 및 문제 해결

**언제 보는가**: 코드 작업 후 검증 시, 오류 해결 시

## 🎯 Reference 사용법

### 상황별 빠른 참조

#### "지금 프로젝트 상태가 어떻지?"
→ [프로젝트 현재 상태](project-status.md)
- 한눈에 보는 현황 파악
- 품질 점수 및 이슈 확인

#### "이 코드가 제대로 작동하는지 확인하려면?"  
→ [검증 명령어](verification-commands.md)
- `npm run verify:parallel` 실행
- 영역별 상세 검증 방법

#### "이 파일이 어디에 있지? 새 파일은 어디에 만들지?"
→ [프로젝트 구조](project-structure.md)  
- 파일 위치 및 명명 규칙
- 기능별 연관 파일들

#### "자동화가 어떻게 돌아가고 있는지 궁금해"
→ [자동화 시스템](automation-systems.md)
- 16개 Agent 자동 활성화
- CI/CD 파이프라인 현황

```
- 컴포넌트 개발 중 → 검증 명령어로 품질 확인
- 데이터베이스 작업 중 → 자동화 시스템에서 관련 스크립트 확인

```

## 💡 효과적인 활용 패턴

### 일일 작업 루틴
1. **작업 시작**: [프로젝트 상태](project-status.md) 확인
2. **개발 중**: [프로젝트 구조](project-structure.md)로 파일 위치 확인
3. **작업 완료**: [검증 명령어](verification-commands.md)로 품질 검증
4. **주간 점검**: [자동화 시스템](automation-systems.md)으로 시스템 현황 파악

### 문제 해결 패턴
1. **문제 발생** → [검증 명령어](verification-commands.md)로 원인 파악
2. **해결 방법 검색** → [프로젝트 구조](project-structure.md)에서 관련 파일 찾기
3. **상태 확인** → [프로젝트 상태](project-status.md)에서 전체 영향 파악
4. **자동화 확인** → [자동화 시스템](automation-systems.md)에서 관련 도구 활용

### 새 팀원 온보딩 패턴
1. **프로젝트 이해**: [프로젝트 상태](project-status.md) → 전체 현황 파악
2. **구조 파악**: [프로젝트 구조](project-structure.md) → 코드 조직 이해
3. **도구 활용**: [검증 명령어](verification-commands.md) → 품질 관리 방법
4. **시스템 이해**: [자동화 시스템](automation-systems.md) → 개발 환경 파악
