# 🎓 Tutorial - 학습 중심 가이드

## 📌 문서 관리 지침
**목적**: AI가 단계별 학습 경로를 통해 프로젝트 개발 능력을 체계적으로 습득  
**대상**: 학습 경로 안내가 필요한 AI 또는 튜토리얼 순서를 확인하려는 경우  
**범위**: 3개 핵심 튜토리얼의 목적, 순서, 학습 성과, 다음 단계 안내 포함  
**업데이트 기준**: 새 튜토리얼 추가, 학습 경로 변경, 연관 문서 구조 변화 시  
**최대 길이**: 4000 토큰 (개요 문서이므로 간결함 유지)  
**연관 문서**: 모든 tutorial/ 문서들, [How-to 가이드](../how-to/), [Reference](../reference/)

## ⚠️ 금지사항
- 각 튜토리얼의 상세 내용 중복 금지 (→ 개별 파일에서 관리)
- 기술적 구현 세부사항 포함 금지 (→ 개별 튜토리얼로 이관)
- 프로젝트별 특화 내용 확장 금지 (일반적 학습 가이드에 집중)

*디하클 프로젝트를 단계별로 학습하며 실제 개발 능력을 키우세요*

## 📋 튜토리얼 목록

### 1. [프로젝트 빠른 시작](01-quick-start.md) ⏱️ 2분
**"30초 만에 프로젝트 파악하기"**

- 프로젝트 기본 정보 및 핵심 규칙
- 16개 서브에이전트 시스템 개요  
- 첫 작업 시작 전 필수 체크리스트
- 다음 읽을 문서들 순서

**완료 후**: 프로젝트의 전체적인 구조와 규칙을 이해하고 작업할 준비가 됩니다.

---

### 2. [첫 번째 작업 시작하기](02-first-task.md) ⏱️ 15-20분
**"실제로 기능을 추가해보기"**

- 데이터베이스 테이블 생성 (RLS 정책 포함)
- API 엔드포인트 개발 (인증 패턴 적용)
- React 컴포넌트 구현 (shadcn/ui 사용)
- 전체 검증 과정 실습

**완료 후**: 간단한 메모 기능을 통해 전체 개발 워크플로우를 경험합니다.

---

### 3. [일반적인 패턴들](03-common-patterns.md) ⏱️ 10-15분  
**"프로젝트의 핵심 패턴 익히기"**

- 인증, UI, 데이터베이스, 에러 처리 패턴
- 환경변수, 테스팅, 성능 최적화 패턴  
- 반응형 디자인 및 검증 패턴
- Git 워크플로우 및 코딩 컨벤션

**완료 후**: 일관된 코딩 스타일로 품질 높은 코드를 작성할 수 있습니다.
