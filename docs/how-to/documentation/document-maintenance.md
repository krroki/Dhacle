# 🔄 문서 유지보수 시스템

## 📌 문서 관리 지침
**목적**: 문서 시스템의 지속적 품질 관리를 위한 정기적 점검 및 유지보수 체계 제공  
**대상**: 문서 관리 담당 AI, 정기 점검 수행하는 AI  
**범위**: 주기별 점검 체크리스트, 문서 품질 관리, 정리 기준  
**업데이트 기준**: 문서 시스템 구조 변경 시, 품질 기준 변경 시 즉시 업데이트  
**최대 길이**: 3500 토큰  
**연관 문서**: [문서 배치 가이드](document-placement-guide.md), [문서 작성 가이드](../../CLAUDE.md)

## ⚠️ 금지사항

## 🔧 문서 정리 기준

### 📋 통합 대상 문서
```yaml
조건:
  - 토큰 수 < 1000 AND 업데이트 빈도 < 월 1회
  - 내용 중복도 > 50% AND 참조 빈도 < 주 1회
  - 유사 주제 문서 3개 이상 AND 각각 토큰 < 1500

처리 방법:
  1. 핵심 내용 추출
  2. 메인 문서에 통합
  3. 개별 문서 아카이브
  4. 리다이렉트 링크 설정
```

### 📋 분리 대상 문서
```yaml
조건:
  - 토큰 수 > 설정 제한 + 20%
  - 여러 주제 혼재 AND 참조 패턴 다양
  - 업데이트 빈도 > 주 3회 AND 변경 부분 분산

처리 방법:
  1. 주제별 내용 분류
  2. 새 문서 생성
  3. 메인 문서에 링크 설정
  4. 관련 문서 상호 참조
```

### 📋 아카이브 대상 문서
```yaml
조건:
  - 마지막 업데이트 > 6개월 AND 참조 빈도 < 월 1회
  - 더 이상 유효하지 않은 정보 > 70%
  - 대체 문서 존재 AND 이전 가치 < 10%

처리 방법:
  1. 아카이브 폴더로 이동
  2. 메인 문서에서 링크 제거
  3. 대체 문서 링크 추가
  4. 이전 기록 보존
```

## 🚨 문제 상황 대응

### ⚠️ 문서-코드 불일치 발견
```yaml
즉시 조치:
  1. 해당 문서 "⚠️ 업데이트 중" 표시 추가
  2. 실제 코드 상태 확인
  3. 정확한 정보로 문서 수정
  4. 검증 후 표시 제거

근본 원인 분석:
  1. 코드 변경 시 문서 업데이트 누락 원인 파악
  2. 자동화 가능한 부분 식별
  3. 체크리스트 보완

재발 방지:
  1. 해당 패턴을 mistake-patterns.md에 추가
  2. 관련 작업 시 필수 체크리스트 강화
  3. 자동 검증 스크립트 개선
```

### ⚠️ 중복 내용 발견
```yaml
즉시 조치:
  1. 중복 내용 식별 및 분류
  2. 메인 위치 결정 (가장 적절한 문서)
  3. 다른 위치에서는 링크로 교체
  4. 일관성 검증

품질 개선:
  1. 중복 발생 원인 분석
  2. 문서 역할 경계 재정의
  3. 작성 가이드 보완

모니터링 강화:
  1. 정기 점검 시 중복 검사 강화
  2. 새 문서 작성 시 중복 사전 확인
  3. 자동 중복 감지 스크립트 활용
```

### ⚠️ 문서 비대화 발견
```yaml
즉시 조치:
  1. 토큰 제한 초과 문서 식별
  2. 핵심 내용과 부가 내용 분리
  3. 부가 내용을 적절한 문서로 이관
  4. 메인 문서에 참조 링크 설정

구조 개선:
  1. 문서 분할 기준 재검토
  2. 하위 문서 생성 또는 기존 문서 활용
  3. 내비게이션 구조 최적화

예방 조치:
  1. 토큰 제한 모니터링 강화
  2. 작성 중 실시간 길이 체크
  3. 정기 점검 빈도 증가
```
